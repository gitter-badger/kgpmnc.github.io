How to update the daily blogger?

  • Fork the repository https://github.com/kgpmnc/blog-source-code and clone the fork. Remember clone your fork not the original repo!
  • Create a new branch.
  • Go into the content directory.
  • Create a markdown file with the date. Use the pattern used earlier.
  • Look inside the previous files and write Title, Date, Tags, Category, Slug, Author, Summary. Look other files for help.
  • If done, navigate back to the repositoy, and run this in the terminal.
$ make html
  • If there is any error, you must have not followed the format. Else, do
$ git add content/
$ git commit -am <commit message containing the date>
$ git push origin <branch name>

Then go to the repository and create the Pull Request. People who were also present in the class are required to verify the information and merge the PR.

  • Now go inside the output directory and copy everything. That is the new site ready to be published.
  • Fork the repository https://github.com/kgpmnc/blog and clone the fork. This is the source code of the web page.
  • Create a branch.
  • Paste all the files that you copied from the Output directory. Yes, replace the files! Create the PR. Same criteria for merging.
  • And that information will go live on the blog.
  • Note : If a change is required, do the change in the PR for the repo blog-source-code. Do not manually change the code for blog. Let it be generated by Pelican.

And it is done!! ta-da!
